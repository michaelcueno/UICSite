Main Window

This is the main window that the user sees when walking into the kitchen. Literally, it's a window. We wanted to implement the app as if it was a natural part of the kitchen. Thus the main screen is meant to model the kitchen window where each user is a blind in the window. In each blind, you can see immediately the information that is most important to you. There are updates for emails, twitter and rss feeds and alerts which could pertain to traffic or weather alerts.


On the bottom we have the icons for the different apps that will load inside the window. (icons made by Matt)

Weather App

The weather application loads inside the main window. It shows the weather outside and offers the user a swiper that changes the current hour that the weather app is looking at. Different weather backgrounds

As the user swipes across the hours, the weather is updated and the background also changes to reflect the weather that current hour is on. This gives a more interactive feel to the weather, allowing the UI to melt away and just give the user what he/she wants, the weather!



Music App

The music player was designed (like everything else) with a touch interface in mind. The available tracks appear in the left hand sidebar under the heading 'library'. When a user taps on a song in the library, it is loaded into the next song queue as seen in the image below. The album art will load in the main area, and the user can interact with the song with the player on the bottom.

A song is loaded into the player, the user can seek using the seeker, play, pause or go to next or previous songs.

Clock App

The clock application was implemented to specifications laid out in the final design. The main focus in any clock application should be the clock! That is why it is in the center and the largest thing in the app. The four panels around it show the different 'time-related' functions that can be performed.
The Timer features three spinners that set hours minutes and seconds, which can then be paused, stopped an cleared. When the timer reaches zero, an alarm sounds. There is a drop down menu to choose a specific sound.

The Countdown will count down to a specific data that the user specifies. Alarms are more or less what is expected. There are a number of alarms that can be turned on or off, and the user can add more by tapping the add more, in which case a new alarm will be added to the list.
The on off button is implemented as an easy to understand toggle switch
The stopwatch features a start, stop and lap buttons which act like any stopwatch should. (Not implemented)

Home Automation App

The home automation app's most important feature is the thermostat control. This is implemented as a slider on the thermostat that is easily understood as hot and cold based on the color. The temperature is displayed in the center along with picures indicated if the thermostat is heating or cooling.
On the bottom of the screen we have webcams so the user can see what's going on in their home from one location.


Garage Door

The garage door on the left, upper corner, is a pictoral indication of whether the garage is open or closed. A simple push button toggles the garage door. This was chosen instead of a 'open' - 'close' button because of most peoples norms with garage door openers. Most people expect to have one button openers that both open and close the door.

Leave Button

The leave button is for when you want to leave the house and lock it down. Pushing the leave button opens the garage, unlocks the door and pauses any music playing in the background. Then a count down starts for you to exit the home. Once the count down ends, the door will lock and the thermostat will lower a few degrees. The garage door remains open, since it would be difficult to time that correctly, you wouldn't want to have trouble starting your car and then have it close on you! So it is up to you to close the garage door from the car when you leave.
