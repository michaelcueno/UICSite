Installation Guide

Installing Qt

Install Qt 5.0 from this link: http://qt-project.org/downloads. It contains the libraries and the IDE 'Qt Creator'. You should need a compiler (Visual Studio, XCode, GCC, ...) if you stick with QML programming.

Download WeatherStation

After downloading, extract and open WeatherStation.qmlproject in Qt Creator. Then, in the 'Project' tab choose an appropriate build environment. Choose the build environment on the bottom left hand corner and then click the green arrow to run the project in a qmlscene.

Alternatively, locate your qmlscene executable and enter 

            qmlscene WeatherStation.qml

FILE STRUCTURE

-WeatherStation.qml              // Main qml file
-WeatherStation.qmlproject       // Project file, open this in qt creator if desired 
+qml/                            // Contains all qml files 
  +Dashboard/                    // Contains qml files related to the 'dashboard' or settings page 
+images/                         // Contains all images used in program
+javascripts/ 
  Tr.js                          // Translation javascript, used to translate text (not working yet)

known Bugs

Unfortunately, the short developing period for this app forced me to release without addressing a troublesome bug. Namely, the cosmic_unit does not scale correctly. It is based on the index to the swiper wich is 0 at midnight (the reason I didn't catch this one is because I typically worked on the project for an hour before bed, aka midnight..) and so as the day goes on the cosmic_unit is offset more and more until by 5-6 o'clock it changes from sun to moon directly in the center of its orbit. I'll address this soon and continue to make updates

Warnings

In order to get the hourly data, I used weatherUndergrounds API, which only allows developers using their free service to call the API 10 times an hour. So when testing, keep this in mind, every time you add a new city, that is a new call as well as every time you launch the applicaiton. I don't see this being a huge problem, but in case you go over the limit, you can synthesize the data by going into the code.

In the WeatherStation.qml file on the bottom is a function called init(), in it you will see comments directing you how to toggle the data from the API to synthesized.
